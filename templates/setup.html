Step-by-step guide

Setup on Portainer

Deploy No-IP Renewer on your Synology NAS using Portainer CE. Takes about 10 minutes.

01

Fork the GitHub Repository

Create your own copy of the code

1
Open github.com/W00DFISH/noip-renewer-docker and click "Use this template" → "Create a new repository".
2
Give it a name like noip-renewer-docker. Set visibility to Private to keep your credentials safe if you ever edit the files.
3
Click "Create repository". Copy the repository URL — you'll need it in Step 3.
02

Install Portainer on Synology

Skip this step if Portainer is already running

1
Open Container Manager on your Synology DSM (Package Center → Container Manager).
2
Go to Project → Create, paste this compose config to install Portainer:
portainer-install.yml
services: portainer: image: portainer/portainer-ce:latest container_name: portainer ports: - "9443:9443" volumes: - /var/run/docker.sock:/var/run/docker.sock - portainer_data:/data restart: unless-stopped volumes: portainer_data:
3
Access Portainer at https://nas-ip:9443 and create your admin account.
03

Create Stack in Portainer

Deploy directly from your GitHub repo

1
In Portainer, go to Stacks → Add stack. Give it the name noip-renewer.
2
Select the "Repository" tab. Fill in:
• Repository URL: your forked repo URL
• Reference: main
• Compose path: docker-compose.yml
3
Enable "Automatic updates" if you want Portainer to watch for git changes (optional — you can also use the in-app update button).
4
Click "Deploy the stack". Portainer will pull the code, build the image, and start the container.
First build takes 3–5 minutes as it installs Chromium. Subsequent starts are instant.
04

Environment Variables

The only variables needed in docker-compose — everything else is configured in the web UI

Variable Required Description
CONTAINER_NAME Optional Container name for self-restart on update. Default: noip-renewer
GITHUB_REPO Optional Your forked repo URL for self-update. Default: https://github.com/W00DFISH/noip-renewer-docker
All No-IP credentials (email, password, TOTP key) and schedule settings are configured inside the web UI and stored in a Docker volume — never in docker-compose.
05

Open the Web UI & Configure

Enter your credentials and set your schedule

1
Open http://nas-ip:7895 in your browser.
2
Go to ⚙️ Config → fill in your No-IP Email, Password, and TOTP Key (if using 2FA).
3
Set your Schedule: enable it, choose your GMT offset, run frequency, and hour. Click Save schedule.
4
Click ▶ Run Now to test. Watch the live log — you should see ✅ Login successful! followed by confirmation of each host.
06

Getting Your TOTP Key (2FA users only)

How to find the 16-character key from No-IP

1
Log in to No-IP → Account → Security → Two-Factor Authentication.
2
When setting up 2FA, next to the QR code there is a plain-text key like JBSWY3DPEHPK3PXP. That is your TOTP Key.
3
Paste it into the TOTP Key field in the web UI.
If you already set up 2FA and didn't save the key — disable then re-enable 2FA to get a new one. There's no way to retrieve an existing key.

You're all set ✓

The renewer will run automatically on your schedule and confirm all expiring hosts. You'll only be notified if something goes wrong.
